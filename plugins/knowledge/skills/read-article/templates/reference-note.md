## 求證結果

針對文章「承重」的核心主張上網查證，逐條標註結論。不必查證每一句，只查少數關鍵且可驗證的主張。

| 核心主張 | 結論 | 依據 |
| -------- | ---- | ---- |
| 文章說 X | 已證實 / 有爭議 / 查無實據 | [來源標題](url)，一句話說明 |
| 文章說 Y | 有爭議 | [來源標題](url)，主流看法為… |

> 若某個承重主張被查證為**錯誤或有重大爭議**，務必在回報時主動點出（見 SKILL.md 的「需要時才問」規則）。

---

## 結論與洞見

- 讀完後最值得記住的 2–4 點，用自己的話寫，而非照抄摘要。
- 點出文章的盲點、適用邊界，或與其他來源的矛盾。

---

## 如何應用

> 僅在文章具學習意義時撰寫；低價值文章略過本節。

將洞見對接到具體的個人脈絡，並用 `[[wiki-link]]` 串到對應的目標或專案筆記：

- **生活**：可落地的具體行動。
- **工作 / 專案**：例如「此筆記法可用於 [[warra]] 的開發紀錄」「對接目標 [[o2-ai-masters-application]]」。
- 每條應用盡量具體、可執行，而非泛泛而談。
