## 求證結果

針對文章「承重」的核心主張上網查證，逐條標註結論（含真假**與是否現行**）。不必查證每一句，只查少數關鍵且可驗證的主張。

| 核心主張 | 結論 | 現行更正（若已過時） | 依據 |
| -------- | ---- | -------------------- | ---- |
| 文章說 X | 已證實 / 有爭議 / 查無實據 / 已過時 | — | [來源標題](url)，一句話說明 |
| 文章說 Y | 已過時 | 現況為 Z（截至今日） | [來源標題](url)，主流看法為… |

> 文章發布日視為警訊：寫作當下正確 ≠ 今日仍正確（版本、價格/規格、工具更名或停用、標準/法規更新、結論被推翻）。任一承重主張被查為**錯誤、重大爭議或已過時**，務必在回報時主動點出。
> 若本文更新/牴觸 vault 既有筆記中的事實，於回報點名該過時筆記，交由使用者決定是否更正（不逕自改動）。

## 結論與洞見

- 讀完後最值得記住的 2–4 點，用自己的話寫，而非照抄摘要。
- 點出文章的盲點、適用邊界，或與其他來源的矛盾。

---

## 如何應用

> 僅在文章具學習意義時撰寫；低價值文章略過本節。

將洞見對接到具體的個人脈絡，並用 `[[wiki-link]]` 串到對應的目標或專案筆記：

- **生活**：可落地的具體行動。
- **工作 / 專案**：例如「此筆記法可用於 [[warra]] 的開發紀錄」「對接目標 [[o2-ai-masters-application]]」。
- 每條應用盡量具體、可執行，而非泛泛而談。
